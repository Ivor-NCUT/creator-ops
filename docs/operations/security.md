# 安全说明

- 单次部署只服务一个公司；权限按 Owner、Admin、HR、Finance、Manager、Member、Viewer 区分。
- 关闭公开注册，初始 Owner 只可在空库创建；演示 seed 不能用于生产。
- 使用 HTTPS，三份数据库/认证/自动化密钥彼此独立，定期轮换并只保存在部署密钥系统。
- PostgreSQL 不映射公网端口。数据库备份包含人员、薪资、合同等敏感数据，必须加密、异机保存并限制访问。
- CSV 导入默认 dry-run；写入需显式 `ALLOW_IMPORT_WRITE=true`，且整批事务失败回滚。导出文件在仓库外处理并在完成后安全删除。
- 审计日志记录关键业务操作。管理员应定期停用离职成员、检查 Owner/Admin/HR/Finance 角色和异常自动化运行。
- 文件上传尚未实现；`FileMetadata` 不等于文件安全扫描。未来接入对象存储时需增加私有桶、短时签名 URL、类型/大小校验和恶意文件扫描。

发现漏洞请私下联系仓库维护者，不要在公开 Issue 中粘贴真实记录、密钥或可利用细节。
