# 升级

1. 阅读目标版本发布说明并先备份。
2. 在测试环境用备份副本执行升级和核心流程验证。
3. 拉取明确的版本标签，而不是不固定的开发分支。
4. `docker compose up -d --build`；独立 migration job 成功后 Web 才启动。
5. 检查 `docker compose logs migrate`、健康检查、登录、看板和自动化运行记录。

Prisma 迁移只向前执行。不要修改已经发布的迁移文件，也不要用 `db push` 代替生产迁移。迁移失败时停止发布并保留日志；根据发布说明回退应用或从升级前备份恢复到新建空库，不要在原库上反复清理。
